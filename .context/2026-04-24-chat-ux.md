# Chat UX Improvements

## What was done
- Added an inline loading indicator for assistant replies (three animated dots) that appears immediately after user submit.
- Reworked chat message rendering so the assistant loader is replaced in place with the final response.
- Improved assistant message readability by splitting output into paragraphs based on line breaks.
- Improved chat message spacing, line-height, and visual separation between user and assistant blocks.
- Updated `SYSTEM_PROMPT` to reduce repetitive phrasing and encourage shorter, conversational 1-3 paragraph responses.

## Files changed
- `src/layouts/BaseLayout.astro`
- `src/lib/ai.ts`

## Notes on behavior changes
- Submitting a question now instantly shows a subtle typing loader in the assistant message area.
- User messages are visually highlighted using the accent palette for clearer role separation.
- Assistant responses render as short paragraph blocks with spacing between paragraphs when line breaks are present.
- Sidebar made static so it stays visible during page scroll.
- Minimal styling changes applied: sticky positioning and viewport-tied height.
- Loader visibility fix: applied global selectors to loader classes and set loader color to accent for stronger contrast on dark background.
- Root cause: loader nodes are created dynamically in JS, and scoped Astro CSS selectors were not matching those runtime elements.
